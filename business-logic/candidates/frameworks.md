# Frameworks - 思维模型/决策框架/推理方法

从《段永平投资问答录-上（商业逻辑篇）》中提取的所有可迁移思考结构。

- id: f01
  title: 做对的事情 vs 把事情做对
  type: framework
  source_chapter: 第1节/第3节/第7节
  source_quote: |
    "做对的事情，把事情做对。知道要做对的事情（或者说，知道什么样的事情不该做），然后去追求高效率把事情做对的公司。"
  summary: |
    段永平核心决策框架的二分法。"做对的事情"是方向判断（道），属于大是大非问题，只有对错无程度；"把事情做对"是执行效率（术），有高低之分，是学习过程。先确定方向正确，再追求执行效率。做对的事情发现错了就马上停，不管多大代价都是最小代价。二者不可颠倒——方向错了，执行越好结果越差。
  tags: [决策框架, 道-术二分, 核心方法论]

  title: Right Business / Right People / Right Price 三维过滤框架
  type: framework
  source_chapter: 第2节
  source_quote: |
    "right business，right people，right price. 对的生意说的就是生意模式，对的人指的就是企业文化。Price没有那么重要，business和people最重要。"
  summary: |
    投资决策的三维过滤器。按优先级排序：商业模式（Right Business）排第一，企业文化/管理层（Right People）排第二，价格（Right Price）排第三。商业模式决定长期现金流，企业文化决定能否维护商业模式，价格是时间的朋友可以降低其重要性。三个维度同时满足为完美，但只要前两个成立且有合理价格即可出手。
  tags: [投资框架, 三维过滤, 优先级排序]

- id: f03
  title: Stop Doing List 逆向排除框架
  type: framework
  source_chapter: 第7节/第3节
  source_quote: |
    "好的公司都一定是有一个长长的'Stop doing list'，就是'不做的事情'。秘诀不是做了什么而是不做什么。"
  summary: |
    通过"不做什么"来定义"做什么"的逆向推理框架。段永平认为成功的秘诀在于建立一张"不做清单"，而非追逐机会清单。不做的清单包含两类：一是谁都不该做的（如欺骗），二是与公司使命愿景结合的。每个不做的事项背后都有故事、道理和逻辑。通过排除法逼近正确方向，而非正面列举。
  tags: [逆向思维, 排除法, 决策框架]

- id: f04
  title: 平常心-回归本源思考
  type: framework
  source_chapter: 第3节
  source_quote: |
    "平常心其实就是在任何时候，尤其是在有诱惑的时候，能够排除所有外界的干扰，回到事物的本质（原点），辨别事情的是非与对错，知道什么是对的事情。"
  summary: |
    面对诱惑和压力时的思维方法。核心机制：遇到问题时先排除所有外界干扰（诱惑、恐惧、压力），回到事物的本源/原点去思考，辨别是非对错。这不是消极心态，而是理性的思考习惯——关注事物本质而非表象。在错误的道路上无法有平常心，平常心需要以"做对的事情"为前提。
  tags: [思维方式, 回归本质, 理性思考]

- id: f05
  title: 10-20年长期倒推法
  type: framework
  source_chapter: 第2节
  source_quote: |
    "投资人每次做投资决定时如果想的是10年20年的事情，最后的结果很难不好，不然就难说了。要找到自己能想清楚10年20年的公司是非常不容易的。"
  summary: |
    决策时以10-20年为尺度进行倒推的思考方法。每次做投资或经营决定时，先想10年20年后会怎样。这能过滤掉短期噪音和诱惑。段永平判断公司时常常想10年后这家公司能赚多少钱。凡事能往前（后）想几年，往往难度会大幅度下来。这种长周期视角本身就是一种护城河。
  tags: [时间尺度框架, 长期思维, 逆向推演]

- id: f06
  title: 差异化分析框架
  type: framework
  source_chapter: 第2节/第4节
  source_quote: |
    "差异化指的是产品能满足用户的某个或某些别人满足不了的需求。能够长期维持的差异化就是护城河。"
  summary: |
    评估商业模式好坏的结构化分析方法。差异化的定义：用户需要但竞争对手满足不了的东西。分析路径：先识别用户真实需求，再看竞争对手是否满足，未满足的就是差异化空间。差异化的程度决定行业盈利能力——差异化越小（如航空、光伏），越难赚钱；差异化越大（如白酒、奢侈品），护城河越宽。差异化是动态的，当大家都有的就变成基本需求。
  tags: [分析框架, 商业模式评估, 护城河识别]

- id: f07
  title: 滚雪球模型（长坡厚雪）
  type: framework
  source_chapter: 第1节
  source_quote: |
    "什么公司是在长长的坡上滚着厚厚的雪呢？长长的坡其实不光指的是行业，而是包括企业本身能否长跑，所以企业文化很重要。"
  summary: |
    评估企业长期价值的空间-质量二维模型。长坡=行业空间大+企业能长跑（需要企业文化支撑）；厚雪=利润率高+净现金流充沛。二者缺一不可：短坡厚雪无法形成大雪球，长坡薄雪滚不出大雪球。坡不仅是行业属性，更取决于企业自身的长跑能力（文化决定）。
  tags: [评估模型, 空间-质量二维, 长期主义]

- id: f08
  title: "造钟人 vs 报时人" 判断框架
  type: framework
  source_chapter: 第1节
  source_quote: |
    "乔布斯同时还是一个非常好的'造钟人'，库克其实就是乔布斯最伟大的发明（发现）之一。"
  summary: |
    评估管理层是否建立可持续制度的框架。报时人=个人能力极强但离开后组织就崩溃的领导者；造钟人=建立制度、文化和传承体系，让组织超越个人而持续运转的领导者。判断方法不是看领导者多强，而是看其离开后组织能否持续——乔布斯建了苹果文化（钟），库克只是执行者。投资应寻找造钟人领导的公司。
  tags: [管理层评估, 制度判断, 可持续性]

- id: f09
  title: 未来现金流折现思维
  type: framework
  source_chapter: 第2节
  source_quote: |
    "投资确实就是买的未来的现金流（折现），未来现金流（折现）最大的保障就是'商业模式'，商业模式里最强的就是垄断，或者叫'护城河'。"
  summary: |
    一切投资判断的底层思维框架。不关注股价波动，不关注短期业绩，而是追问：这家公司在其整个生命周期内能产生多少净现金流？具体操作：判断生意模式好不好（决定现金流的大小和持续性），判断企业文化好不好（决定生意模式能否维持），给个5-10年看增长。不需要精确计算，而是方向性判断——毛估估即可。
  tags: [估值框架, 第一性原理, 现金流思维]

- id: f10
  title: 焦点法则（Focus）
  type: framework
  source_chapter: 第4节/第7节
  source_quote: |
    "一个企业家应将80%的精力放在20%的事情上，这20%的事情能给你带来80%效益。鸡蛋就放在一个篮子里头，这样我才看得住。"
  summary: |
    资源分配的优先级框架。核心逻辑：每个企业的资源（人力、物力、财力）都是有限的，必须高度聚焦在最有价值的事情上。企业如跳水运动员，动作越少越好。产品品种越少，研发效率越高、质量一致性越好、成本越低、渠道库存压力越小。聚焦的人或公司最后会变得很厉害，但做到"少"非常难。
  tags: [战略框架, 资源聚焦, 少即是多]

- id: f11
  title: 敢为天下后，后中争先
  type: framework
  source_chapter: 第4节/第6节
  source_quote: |
    "所谓差异化指的就是客户需要但别人未能满足的东西。没有差异化的'敢为天下后'是无法生存的。"
  summary: |
    市场进入策略的结构化推理。前提条件：必须有差异化的能力。推理路径：先观察市场（看别人做什么、什么好卖），确认自己能提供差异化的产品/服务，然后集中优势兵力快速切入。不追求先发优势（培育市场代价太大），而是在确认需求后再出手。关键不是"敢为天下后"本身，而是"后中争先"——进入后做到最好。
  tags: [市场进入策略, 后发优势, 竞争策略]

- id: f12
  title: 利润之上追求的决策过滤器
  type: framework
  source_chapter: 第1节/第3节
  source_quote: |
    "利润之上的追求指的是把消费者需求放在公司短期利益前面。多数公司碰到问题时讨论的都是有没有钱赚的问题，而有利润之上追求的公司碰到问题时可能会先问一句，这是对的事情吗？"
  summary: |
    面对利益诱惑时的决策过滤机制。当面临商业决策时，不先问"赚不赚钱"，而是先问"这是对的事情吗？这是该赚的钱吗？"有利润之上追求的公司看到事物本质更容易，更不容易被短期诱惑偏离大方向。这不是充分条件（有追求不一定是好企业），但时间越长与一般企业的差别越大。
  tags: [决策过滤, 价值观驱动, 长期导向]

- id: f13
  title: 时间降维法（时间降低价格重要性）
  type: framework
  source_chapter: 第2节
  source_quote: |
    "right business最重要，因为right people在right business里能充分体现出来。right price则没那么重要，因为时间可以降低其重要性。"
  summary: |

  tags: [时间维度思维, 简化决策, 长期主义]

- id: f14
  title: 概率思维（好公司犯错的概率更低）
  type: framework
  source_chapter: 第1节
  source_quote: |
    "知道'概率'的意思吗？真的知道吗？巴菲特买错过很多只股票，每一只错误的股票都可以被称为'失败'，巴菲特'失败'吗？"
  summary: |
    用概率而非确定性来思考企业质量的框架。好公司的定义不是不犯错，而是犯原则性错误的概率远低于一般公司，发现错误更早、改得更快。判断企业不是看单次结果，而是看长期概率分布。好企业文化让犯错的概率降低、纠错的概率提高，但不保证每次都对。投资是概率游戏，不是确定性游戏。
  tags: [概率思维, 风险管理, 统计思维]

- id: f15
  title: 原点回归思考法（本分思维）
  type: framework
  source_chapter: 第3节
  source_quote: |
    "本分就是要做对的事情和要把事情做对。平常心就是回到事物本源的心态，也就是要努力认清什么是对的事情，认清事物的本质。"
  summary: |
    遇到任何问题都回到原点重新思考的方法。本分=做对的事情+把事情做对。遇到问题时：1）隔离外在压力和诱惑，保持平常心态；2）回到事物的本源去思考什么是对的；3）求责于己而非指责他人；4）不赚人便宜。本分是检视自己的工具，不是照妖镜。难在"明知是错的事情还去做"（因为短期诱惑），所以需时刻提醒。
  tags: [思维方式, 回归本质, 自省框架]

- id: f16
  title: 马和骑师模型（商业模式 vs 管理层）
  type: framework
  source_chapter: 第2节
  source_quote: |
    "商业模式像马，管理层加企业文化像骑师，经营结果就是比赛结果。不过企业经营不像赛马那么短时间而已。再好的车手也很难开好一辆烂车。"
  summary: |
    理解商业模式和管理层关系的类比推理模型。马（商业模式）是基础——再好的骑师也骑不好一匹烂马；骑师（管理层+企业文化）是放大器——好马配好骑师才能赢。当强调人时，表示商业模式已经看过了。二者不知道哪个更重要，但知道哪个不行时哪个就重要。应用于：评估企业时先看马再看骑师。
  tags: [类比模型, 商业模式评估, 管理层判断]

- id: f17
  title: 从否定面判断法（知道什么不对比知道什么对更容易）
  type: framework
  source_chapter: 第1节/第3节
  source_quote: |
    "很难判断领导层是否'对'，但相对容易判断他们是否'不对'。不对的就不碰，错误的概率就大幅度降低了。什么是不对的没那么难。"
  summary: |
    通过排除法而非确认法来做判断的推理路径。判断"对"很难（需要深度理解和预见），但判断"不对"相对容易（有明显的信号和红线）。应用：判断管理层——看他们做了什么不对的事比看他们做了什么对的事更有效；判断企业——避开有明确问题的企业比找到完美企业更可行。段永平不买的理由往往一两个就够了。
  tags: [排除法, 否定判断, 决策简化]

- id: f18
  title: 消费者导向优先级链
  type: framework
  source_chapter: 第3节/第4节
  source_quote: |
    "消费者导向放之四海而皆准。技巧的东西谁都可以学，但我们之所以成为我们，除了我们一直努力做的事情外，我们不做某些事情的决定也同样重要。"
  summary: |
    商业决策中各导向的优先级排序。消费者导向 > 技术导向 > 市场份额导向 > 利润导向。消费者导向是根本——技术是为产品服务的工具，市场份额是结果不是目标，利润是水到渠成。区分客户和用户（狗粮案例：客户是狗主人，用户是狗），产品设计应以最终用户为准。市场调查只验证直觉，不能替代直觉。
  tags: [优先级框架, 消费者导向, 商业本质]

- id: f20
  title: 欲速不达与足够的最小发展速度
  type: framework
  source_chapter: 第3节/第7节

- id: f21
  title: 护城河验证法（能涨价=有护城河）
  type: framework
  source_chapter: 第2节

- id: f22
  title: 沉没成本止损框架
