# Duan Yongping Skill

一套可直接使用的 AI skill 工具包——把段永平投资问答录中的商业逻辑与投资方法拆成了 15 个可单独触发、也可组合调用的判断模块。

这不是一本书的摘要页。书是这些 skill 的来源，不是首页主角。

## 这套 skills 能解决什么问题

- 想判断一家企业值不值得投资，但不知道从哪里看起
- 分不清"好的生意模式"和"好的产品"的区别
- 不确定自己是在做"对的事情"还是"把事情做对"
- 想建立自己的 Stop Doing List，但不知道该排除什么
- 在集中还是分散、持有还是卖出之间犹豫不决

## 蒸馏了什么

从《段永平投资问答录》（商业逻辑篇 + 投资逻辑篇）中提取了 15 个经过三重验证的 skill，分为两部分：

### 商业逻辑篇（7 个 skill）

- **底层哲学**：`right-things-first`（做对的事情二元决策）、`stop-doing-list`（不为清单构建法）
- **投资总纲**：`right-business-people-price`（投资三要素 B>P>P）、`business-model-evaluator`（商业模式评估）、`culture-judger`（企业文化判断）
- **竞争策略**：`dare-to-follow`（敢为天下后）、`profit-above-pursuit`（利润之上的追求）

### 投资逻辑篇（8 个 skill）

- **投资哲学**：`ownership-mindset`（买股票就是买公司）、`stop-doing-list`（排除法投资纪律）
- **分析方法**：`capability-circle`（能力圈决策）、`good-business`（好生意识别）、`dcf-valuation`（DCF 思维估值）
- **决策框架**：`opportunity-cost`（机会成本决策）、`concentrated-investing`（集中投资）
- **心理管理**：`equanimity-mindset`（平常心投资）

## 效果示例

### 示例 1：看一家公司该不该买

**用户问题**

"有一家公司产品很好，用户增长很快，但还没盈利，值得投吗？"

**skill 如何判断**

- 先触发 `right-things-first`，确认你是在做对的事情还是只是在把事情做对
- 调用 `right-business-people-price`，按 B>P>P 三要素拆解
- 用 `business-model-evaluator` 深入看商业模式：有差异化吗？有护城河吗？有定价能力吗？
- 补充 `ownership-mindset`：如果你是这家公司的全资老板，你会怎么想？

**最终回答示例**

> 先别看用户增长和盈利，先问一个更根本的问题：这门生意的差异化在哪里？如果产品好但没有差异化，用户只是因为补贴或风潮来的，那增长越快未来摔得越重。然后看商业模式：它靠什么阻止竞争者复制？定价能力从哪来？如果这两个问题答不清楚，产品再好也不是好生意。最后代入老板视角：如果你要全资买下这家公司，靠它未来的现金流回本，你愿意出多少钱？如果答案依赖于"继续融资"或"被收购"，那它不是投资标的，是赌注。

### 示例 2：要不要止损卖出

**用户问题**

"我买的一只股票跌了 30%，要不要止损？"

**skill 如何判断**

- 触发 `opportunity-cost`，用机会成本框架统一买卖持有的标准
- 调用 `equanimity-mindset`，检查是不是在情绪驱动下做决策
- 补充 `capability-circle`，确认当初买入时是否真的在能力圈内

**最终回答示例**

> 止损的问题不在"跌了多少"，而在"现在如果手里是现金，你还会不会买这只股票"。如果会，跌了 30% 反而是更好的机会。如果不会，那当初买入的理由已经变了，和你亏了多少没有关系。大多数止损是情绪驱动的：浮亏带来的不适感让你想"结束痛苦"。但投资决策的标准只有一个——机会成本：你手上的钱放在哪里回报最高。抛开买入价，只看当前价值。

## 这些 skill 是怎么生成出来的

这套 skill 使用 **cangjie-skill** 的 RIA-TV++ 流水线生成，整个流程分六个阶段：

1. **整书理解（Adler 分析）**——用 Mortimer Adler 的分析阅读法，对全书做结构、解释、批判、应用四步拆解，产出 `BOOK_OVERVIEW.md`
2. **并行提取**——同时派 5 个专项提取器（框架、原则、案例、反例、术语），从原文中提取候选方法论单元
3. **三重验证筛选**——每个候选必须通过三项检验：书中至少有 2 处独立佐证（跨域）、能回答书中未明说的新问题（预测力）、不是常识（独特性）
4. **RIA++ 构造**——将验证通过的内容按 R（原文引用）/ I（用自己的话重写）/ A1（书中案例）/ A2（未来触发场景）/ E（可执行步骤）/ B（边界与盲点）六个维度结构化
5. **Zettelkasten 链接**——找出 skill 之间的依赖、对比、组合关系，生成 `INDEX.md` 和引用图
6. **压力测试**——为每个 skill 设计包含诱饵题的测试用例，未通过的回炉重做
